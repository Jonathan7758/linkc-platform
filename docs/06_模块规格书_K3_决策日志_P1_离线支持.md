# Part A: K3 决策日志库

## A1. 模块概述

K3负责记录Agent决策的完整生命周期（Context → Decision → Outcome三元组），是阶段三Agent自学习的数据基础。

**关键设计约束**: 日志写入不能阻塞决策管道。Context在决策前同步采集（低成本），Outcome在任务完成后异步补填。

## A3. 数据库Schema

```sql
-- K3: 决策记录表 (TimescaleDB hypertable)
CREATE TABLE decision_records (
    record_id VARCHAR(64) NOT NULL,
    agent_id VARCHAR(64) NOT NULL,
    agent_type VARCHAR(64) NOT NULL,
    decision_type VARCHAR(64) NOT NULL,
    timestamp TIMESTAMPTZ NOT NULL,
    
    -- 三元组
    context JSONB NOT NULL,
    decision JSONB NOT NULL,
    outcome JSONB,                -- 异步补填，初始为NULL
    
    -- 关联
    workflow_instance_id VARCHAR(64),
    orchestration_task_id VARCHAR(64),
    node_id VARCHAR(32) DEFAULT 'default',
    
    -- 阶段三
    reward_signal FLOAT,
    is_training_candidate BOOLEAN DEFAULT FALSE,
    
    PRIMARY KEY (record_id, timestamp)
);

-- 转为TimescaleDB超级表（按天分区）
SELECT create_hypertable('decision_records', 'timestamp',
    chunk_time_interval => INTERVAL '1 day');

-- 索引
CREATE INDEX idx_dr_agent ON decision_records(agent_type, timestamp DESC);
CREATE INDEX idx_dr_override ON decision_records(timestamp DESC) 
    WHERE (outcome->>'human_override')::boolean = true;
CREATE INDEX idx_dr_training ON decision_records(timestamp DESC) 
    WHERE is_training_candidate = true;
```

## A4. 与Agent运行时的集成

```python
# 在A1 Agent运行时框架中集成DecisionLogger

class AgentRuntime:
    def __init__(self, ..., decision_logger: DecisionLoggerInterface):
        self.decision_logger = decision_logger
    
    async def execute_decision(self, task_input):
        # 1. 采集上下文
        context = await self._collect_context(task_input)
        
        # 2. LLM推理
        raw_decision = await self._llm_inference(context)
        
        # 3. 校验
        validation = await self.validator.validate(raw_decision, context, self.agent_type)
        context.validation_passed = validation.valid
        context.validation_errors = [asdict(e) for e in validation.errors]
        
        final_decision = validation.corrected_decision or raw_decision
        
        # 4. 记录决策（异步，不阻塞）
        record = DecisionRecord(
            record_id=generate_id("dr"),
            agent_id=self.agent_id,
            agent_type=self.agent_type,
            decision_type=self._get_decision_type(task_input),
            context=context,
            decision=final_decision
        )
        asyncio.create_task(self.decision_logger.log_decision(record))
        
        # 5. 执行
        if validation.valid:
            result = await self._execute(final_decision)
            # 6. 补填Outcome（通过事件总线异步触发）
            return result
        else:
            return await self._handle_validation_failure(validation)
```

## B3. 缓存策略

| 数据类型 | 缓存策略 | 过期时间 | 大小估计 |
|---------|---------|---------|---------|
| 当前任务列表 | 每次打开APP时全量拉取 | 30分钟 | ~50KB |
| 机器人状态 | 每30秒增量更新 | 5分钟 | ~20KB |
| 待确认事项 | 实时WebSocket + 缓存 | 不过期 | ~10KB |
| 楼层/区域信息 | 首次加载后长期缓存 | 24小时 | ~30KB |
| 用户配置 | 登录时加载 | 7天 | ~5KB |

## B5. 冲突解决策略

```
冲突场景: 离线期间确认了任务A，但在同步前任务A已被另一个训练师取消

解决规则:
1. 时间戳比较: 如果服务器版本更新，以服务器为准
2. 操作类型优先级: cancel > complete > confirm
3. 用户通知: 冲突时弹出提示，让用户决定

具体实现:
- 每条操作携带 optimistic_version (乐观锁版本号)
- 同步时服务端比较版本号
- 版本冲突返回 409 Conflict + 服务端当前状态
- 客户端展示冲突详情，用户选择操作
```
