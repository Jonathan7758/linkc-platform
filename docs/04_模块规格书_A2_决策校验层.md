# 模块规格书: A2 LLM决策校验层 (DecisionValidator)

> 模块ID: A2-VALIDATOR  
> 优先级: P0  
> 预估工时: 2-3天  
> 前置依赖: F1数据模型(已完成), K2治理规则库(可并行)  
> Claude Code 会话数: 2次

---

## 1. 模块概述

### 1.1 职责

在LLM输出和任务执行之间设置四层校验网，确保Agent决策的合法性、一致性和安全性。

### 1.2 在系统中的位置

```
LLM推理输出 → [DecisionValidator] → 执行层
                  │
                  ├─ 校验通过 → 记录日志 → 执行
                  ├─ 可修正   → 自动修正 → 记录日志 → 执行
                  └─ 校验失败 → 记录日志 → 降级到规则引擎 or 人工升级
```

### 1.3 四层校验链

| 层次 | 校验器 | 职责 | 失败后果 |
|------|--------|------|---------|
| L1 | SchemaValidator | JSON结构是否符合预期 | 阻止执行 |
| L2 | ReferenceValidator | 引用的实体是否真实存在 | 阻止执行 |
| L3 | ConstraintValidator | 是否违反治理规则(K2) | 根据规则类型决定 |
| L4 | SafetyValidator | 是否触碰硬编码安全红线 | 无条件阻止 |

```python
# src/validation/decision_validator.py

from typing import Dict, Any, List, Optional, Protocol
from dataclasses import dataclass, field
from enum import Enum


class ValidatorType(Enum):
    SCHEMA = "schema"
    REFERENCE = "reference"
    CONSTRAINT = "constraint"
    SAFETY = "safety"


class DecisionValidatorInterface(Protocol):
    """决策校验器主接口"""
    
    async def validate(
        self,
        decision: Dict[str, Any],
        context: "DecisionContext",
        agent_type: str
    ) -> "ValidationResult":
        """
        执行全链路校验
        
        Args:
            decision: LLM输出的决策JSON
            context: 决策上下文（包含可用资源、环境状态等）
            agent_type: Agent类型（不同类型有不同Schema）
        
        Returns:
            ValidationResult: 校验结果
        
        校验链执行规则:
        - L1-L4依次执行
        - 任一层返回CRITICAL错误则立即中断
        - WARNING级别的问题记录但不中断
        - L3(ConstraintValidator)可产生自动修正建议
        """
        ...
    
    async def validate_partial(
        self,
        decision: Dict[str, Any],
        validators: List[ValidatorType],
        context: "DecisionContext",
        agent_type: str
    ) -> "ValidationResult":
        """仅执行指定的校验器（用于调试或特殊场景）"""
        ...
```

```python
# src/validation/base_validator.py

class BaseValidator(Protocol):
    """校验器基类接口"""
    
    @property
    def validator_type(self) -> ValidatorType: ...
    
    async def validate(
        self,
        decision: Dict[str, Any],
        context: Dict[str, Any]
    ) -> List["ValidationError"]:
        """
        执行校验，返回错误列表
        空列表表示校验通过
        """
        ...


# === L1 Schema校验器 ===

class SchemaValidatorInterface(BaseValidator):
    """
    JSON Schema校验
    
    校验内容:
    - 决策JSON是否包含所有必填字段
    - 字段类型是否正确
    - 枚举值是否在允许范围内
    - 数值是否在合理区间
    """
    
    async def register_schema(
        self, agent_type: str, schema: Dict[str, Any]
    ) -> None:
        """注册Agent类型对应的决策Schema"""
        ...


# === L2 引用校验器 ===

class ReferenceValidatorInterface(BaseValidator):
    """
    实体引用校验
    
    校验内容:
    - robot_id 是否在系统中存在
    - floor_id / zone_id 是否在当前建筑中存在
    - person_id 是否存在且当前在岗
    - task_id 是否存在且状态允许操作
    """
    pass


# === L3 约束校验器 ===

class ConstraintValidatorInterface(BaseValidator):
    """
    治理规则校验
    
    校验内容:
    - 从K2加载当前生效的GovernanceRule
    - 对决策逐条评估规则条件
    - 返回触发的规则及其动作
    
    特殊能力:
    - 可产生自动修正建议（当规则action_type=MODIFY时）
    """
    
    async def load_active_rules(
        self, scope: str, agent_type: str
    ) -> List["GovernanceRule"]:
        """加载当前生效的规则"""
        ...
    
    async def evaluate_rules(
        self,
        decision: Dict[str, Any],
        rules: List["GovernanceRule"]
    ) -> List["RuleEvalResult"]:
        """评估规则"""
        ...


# === L4 安全校验器 ===

class SafetyValidatorInterface(BaseValidator):
    """
    安全红线校验（硬编码，不可配置）
    
    红线规则:
    1. 不允许同时向同一台机器人下发两个任务
    2. 不允许向电量<5%的机器人下发任何非充电任务
    3. 不允许向已报错的机器人下发任务
    4. 不允许向已关闭的区域派遣机器人
    5. 单次调度机器人数量不超过建筑总机器人数的80%
    """
    pass
```

## 3. 实现要点

### 3.1 校验链执行流程

```python
async def validate(self, decision, context, agent_type):
    result = ValidationResult(valid=True, validators_executed=[])
    
    for validator in [
        self.schema_validator,
        self.reference_validator,
        self.constraint_validator,
        self.safety_validator
    ]:
        errors = await validator.validate(decision, context)
        result.validators_executed.append(validator.validator_type.value)
        
        for error in errors:
            if error.severity == ValidationSeverity.CRITICAL:
                result.valid = False
                result.errors.append(error)
                return result  # 立即中断
            elif error.severity in (ValidationSeverity.ERROR,):
                result.valid = False
                result.errors.append(error)
            elif error.severity == ValidationSeverity.WARNING:
                result.warnings.append(error)
        
        # L3可能产生自动修正
        if hasattr(validator, 'get_corrections'):
            corrections = await validator.get_corrections(decision, errors)
            if corrections:
                result.corrected_decision = corrections
    
    return result
```

```python
async def _check_robot_exists(self, robot_id: str) -> bool:
    """通过M3/M4 MCP查询机器人是否存在"""
    try:
        status = await self.robot_mcp.get_robot_status(robot_id)
        return status is not None
    except Exception:
        return False

async def _check_zone_exists(self, zone_id: str) -> bool:
    """通过M1空间MCP查询区域是否存在"""
    try:
        zone = await self.space_mcp.get_zone(zone_id)
        return zone is not None
    except Exception:
        return False
```

### 3.3 ConstraintValidator与K2集成

```python
async def validate(self, decision, context):
    errors = []
    
    # 1. 加载当前生效的规则
    rules = await self.load_active_rules(
        scope=context.get("scope", "system"),
        agent_type=context.get("agent_type", "")
    )
    
    # 2. 逐条评估
    for rule in sorted(rules, key=lambda r: r.priority, reverse=True):
        eval_result = await self._evaluate_single_rule(rule, decision)
        
        if eval_result.triggered:
            severity = {
                RuleActionType.BLOCK: ValidationSeverity.CRITICAL,
                RuleActionType.WARN: ValidationSeverity.WARNING,
                RuleActionType.MODIFY: ValidationSeverity.WARNING,
                RuleActionType.LOG: ValidationSeverity.INFO,
                RuleActionType.ESCALATE: ValidationSeverity.ERROR,
            }.get(rule.action_type, ValidationSeverity.WARNING)
            
            errors.append(ValidationError(
                validator="constraint",
                field="",
                message=eval_result.message,
                severity=severity,
                rule_id=rule.rule_id
            ))
    
    return errors
```
