## 1. 模块概述

### 1.1 职责

为MCP Server层增加WebSocket/SSE实时推送通道，使机器人状态变化能在秒级推送到系统各层。

### 1.2 当前问题

- M3/M4通过HTTP轮询获取机器人状态，最大延迟等于轮询间隔（默认30秒）
- D1数据采集引擎仅支持定时采集，无法响应实时事件
- G4监控API无实时推送，前端靠轮询刷新

### 1.3 改动范围

| 组件 | 改动类型 | 说明 |
|------|---------|------|
| M3 高仙MCP | 增强 | 新增 `realtime_client.py` |
| M4 科沃斯MCP | 增强 | 新增 `realtime_client.py` |
| D1 数据采集引擎 | 增强 | 新增事件驱动采集模式 |
| G4 监控API | 增强 | 新增WebSocket/SSE端点 |
| F5 事件总线抽象 | 新增 | `EventBusInterface` + Redis实现 |

## 2. 接口定义

### 2.1 实时客户端接口 (M3/M4 共用)

```python
# src/adapters/{brand}/realtime_client.py

from typing import AsyncIterator, Callable, List, Optional, Protocol
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum


class RobotEventType(Enum):
    """机器人实时事件类型"""
    STATUS_CHANGED = "robot.status.changed"
    BATTERY_UPDATE = "robot.battery.update"
    BATTERY_CRITICAL = "robot.battery.critical"
    TASK_PROGRESS = "robot.task.progress"
    TASK_COMPLETED = "robot.task.completed"
    ERROR_OCCURRED = "robot.error.occurred"
    POSITION_UPDATE = "robot.position.update"
    OBSTACLE_DETECTED = "robot.obstacle.detected"
    CONNECTIVITY_LOST = "robot.connectivity.lost"
    CONNECTIVITY_RESTORED = "robot.connectivity.restored"


@dataclass
class RobotRealtimeEvent:
    """机器人实时事件"""
    event_id: str
    robot_id: str
    event_type: RobotEventType
    timestamp: datetime
    data: dict = field(default_factory=dict)
    # data示例:
    # STATUS_CHANGED:    {"old_status": "idle", "new_status": "working"}
    # BATTERY_CRITICAL:  {"battery_level": 15, "estimated_remaining_minutes": 20}
    # ERROR_OCCURRED:    {"error_code": "E001", "message": "前方障碍物无法绕行"}
    # POSITION_UPDATE:   {"floor": 3, "x": 12.5, "y": 8.3, "heading": 90}


class RealtimeClientInterface(Protocol):
    """机器人实时通信客户端接口"""
    
    async def connect(self, robot_id: str) -> bool:
        """建立与机器人的实时连接"""
        ...
    
    async def disconnect(self, robot_id: str) -> bool:
        """断开实时连接"""
        ...
    
    async def subscribe(
        self,
        robot_id: str,
        event_types: List[RobotEventType],
        callback: Callable[[RobotRealtimeEvent], None]
    ) -> str:
        """订阅指定事件类型，返回subscription_id"""
        ...
    
    async def unsubscribe(self, subscription_id: str) -> bool:
        """取消订阅"""
        ...
    
    async def get_connection_status(self, robot_id: str) -> dict:
        """获取连接状态"""
        ...
    
    @property
    def connected_robots(self) -> List[str]:
        """当前已连接的机器人ID列表"""
        ...
```

## 3. 实现要点

### 3.1 高仙机器人实时通道

高仙机器人API支持MQTT或WebSocket推送（需确认具体API版本）。实现策略:

```
方案A（优先）: 高仙云平台WebSocket推送
  MCP Server ←WebSocket← 高仙云平台
  优点: 直接获取结构化事件
  
方案B（降级）: 轮询+变更检测
  MCP Server →HTTP轮询→ 高仙API
  本地对比上次状态，检测到变更后生成事件
  将轮询间隔缩短到5秒
```

### 3.2 断连重连策略

```python
RECONNECT_CONFIG = {
    "max_retries": 10,
    "initial_delay_seconds": 1,
    "max_delay_seconds": 60,
    "backoff_multiplier": 2.0,
    "jitter": True,                # 添加随机抖动防止惊群
    "health_check_interval": 30,   # 心跳间隔(秒)
}
```

### 3.3 事件缓冲

断连期间的事件需要缓冲，重连后补发:

```python
EVENT_BUFFER_CONFIG = {
```
